QIQA Form Filler Privacy Policy

Last updated: 10 July 2026

QIQA Form Filler is a Chrome extension for authorised QIQA users. It helps users fill approved online HTML forms with reviewed patient data from the QIQA EHR.

What The Extension Does

The extension can:

  • read structured form-filling data exposed by the QIQA EHR web app;
  • detect normal HTML form fields on an approved target website;
  • show the extracted and reviewed fields to the user;
  • fill matching HTML inputs, textareas, selects, checkboxes, and radio buttons after user action.

The extension does not:

  • submit forms automatically;
  • fill embedded PDFs;
  • bypass login, CAPTCHA, MFA, or anti-bot systems;
  • sell user or patient data;
  • use data for advertising, profiling, creditworthiness, or unrelated purposes.

Data Handled

The extension may handle the following data when used by an authorised QIQA user:

  • patient identifiers, such as name, date of birth, hospital or folder number, ID or passport number, phone number, and email address;
  • health information, such as diagnoses, ICD-10 codes, medications, procedures, clinical summaries, and motivations;
  • clinician or practice details;
  • website form field labels, names, IDs, placeholders, and other HTML form metadata needed to match fields.

How Data Is Used

Data is used only to help the user review and fill approved online forms. The user remains responsible for reviewing the filled form before submitting it.

Storage

The extension uses Chrome session storage to temporarily hold reviewed form fields, detected target form fields, and the approved target URL during the browser session. This is used so the user can move between the QIQA tab and the target form tab.

The extension does not intentionally store patient data long term.

Data Sharing

QIQA Form Filler does not sell, transfer, or share user or patient data with third parties. The extension does not send patient data to advertising networks, analytics platforms, or unrelated services.

Security And Permissions

The extension uses Chrome permissions to:

  • access the active tab after the user clicks the extension;
  • detect and fill form fields on the selected page;
  • verify that the active form page matches the approved target URL;
  • temporarily store reviewed fields during the browser session.

Host permissions are limited to QIQA, local development, Azure staging, and approved demo or target form sites.

User Control

The user must explicitly extract data, detect fields, and fill the form. The extension never submits a form. The user can clear stored session data using the Clear data button in the extension.
